You want:
Parent → runs C logic
Child → runs Python script
Child produces output
Parent captures that output
Parent uses it
This is exactly what pipes + fork + exec are for.

High-Level Idea:
1. Create a pipe
2. fork()
3. Child: Redirect stdout to pipe
          exec() Python script

4. Parent: Read from pipe
        Convert output to usable value
        Continue processing

Visual flow:
Parent Process
     |
   pipe()
     |
   fork()
    /   \
Child   Parent
 |         |
stdout → pipe
exec python
           read(pipe)
           use value

** pipe(pipefd) **
Creates:
pipefd[0] → read end
pipefd[1] → write end


This is the magic line: dup2(pipefd[1], STDOUT_FILENO)
It says:
"Take everything the child prints to stdout and send it into the pipe instead."
so the value from the python script does not go to the terminal, but instead goes into the pipe
then the parent reads it
